# Public-data multiscale immune decoupling analysis in sepsis

This repository contains journal-neutral code, derived tables, final figure source files, and reproducibility metadata for a public-data computational analysis of multiscale immune-state decoupling in sepsis.

## Current release

- Version: `v0.1.5`
- Authors: Xinzhuo Li and Yi Gong
- Corresponding author in the manuscript: Yi Gong
- GitHub repository: https://github.com/LiXinzhuo0425/sepsis-multiscale-immune-decoupling
- Zenodo record: https://doi.org/10.5281/zenodo.20932860

## Scope

- Public bulk transcriptomic cohorts: GSE154918, GSE134347, GSE65682, GSE32707, GSE69063, GSE28750, and GSE54514.
- Public or previously audited read-only cytometry/CyTOF summary layers used for bounded cross-scale interpretation.
- Derived aggregate result tables, final main and supplementary figure files, supplementary tables, source-data tables, and provenance scripts.
- No MIMIC-IV, eICU, restricted ICU EHR records, protected health information, new human samples, wet-lab experiments, or clinical deployment analysis are included.

## Directory map

- `scripts/`: analysis, provenance, and final packaging scripts used for the public-data workflow.
- `supplementary_data/`: formal supplementary tables, figure source-data tables, source mappings, and a consolidated workbook.
- `figures/`: final main Figures 1-5 and formal Supplementary Figure S1 in vector and raster formats.
- `metadata/`: checksums, QA reports, Zenodo upload notes, release manifests, and final submission QA records.
- `DATA_AVAILABILITY.md`: reusable data-availability statement.
- `CODE_AVAILABILITY.md`: code availability and rerun notes.
- `environment.yml` and `requirements.txt`: environment pins used by packaging and figure-building steps.

## Reproducibility boundary

The scripts are retained as transparent analysis provenance. A full rerun requires recreating or supplying the public processed matrices and read-only summary inputs listed in `supplementary_data/csv/` and related audit files. Local absolute paths from the working computer were replaced by placeholders in this public copy.
