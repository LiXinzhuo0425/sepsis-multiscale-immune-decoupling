# Public-data multiscale immune decoupling analysis in sepsis

This repository contains journal-neutral code, derived tables, final figure
source files, and reproducibility metadata for a public-data computational
analysis of multiscale immune-state decoupling in sepsis.

## Scope

- Public bulk transcriptomic cohorts: GSE154918, GSE134347, GSE65682, GSE32707,
  GSE69063, GSE28750, and GSE54514.
- Public or previously audited read-only cytometry/CyTOF summary layers used for
  bounded cross-scale interpretation.
- Derived aggregate result tables, final main and supplementary figure files,
  supplementary workbooks, source-data tables, and provenance scripts.
- No MIMIC-IV, eICU, restricted ICU EHR records, protected health information,
  new human samples, wet-lab experiments, or clinical deployment analysis are
  included.

## Directory Map

- `scripts/`: analysis, provenance, and final packaging scripts used for the
  public-data workflow.
- `supplementary_data/`: formal supplementary tables, figure source-data tables,
  source mappings, and a consolidated workbook.
- `figures/`: final main and supplementary figure outputs in vector and raster
  formats.
- `metadata/`: checksums, QA reports, Zenodo upload notes, release manifests,
  and final figure/package QA summaries.
- `DATA_AVAILABILITY.md`: reusable data-availability statement.
- `CODE_AVAILABILITY.md`: code availability and rerun notes.
- `environment.yml` and `requirements.txt`: Python environment pins used by the
  packaging and figure-building steps.

## Reproducibility Boundary

The scripts are retained as transparent analysis provenance. A full rerun
requires recreating or supplying the public processed matrices and read-only
summary inputs listed in `supplementary_data/csv/S1_main_full_dataset_inventory.csv`
and related audit files. Local absolute paths from the working computer were
replaced by placeholders in this public copy.

## Suggested Citation

Use the versioned DOI generated by the Zenodo record, or the concept DOI for the
